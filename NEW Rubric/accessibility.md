# Accessibility

Accessibility is a foundational dimension of OER quality, ensuring that all learners—regardless of ability, technology, or learning context—can perceive, navigate, and engage with course materials. Accessible OER remove barriers to learning and support more inclusive, flexible educational experiences.

Accessibility is especially important in OER because open materials are designed to be widely shared, reused, and adapted across diverse educational settings. Without intentional attention to accessibility, barriers can be unintentionally replicated or amplified as resources are redistributed. Designing for accessibility from the outset helps ensure that OER remain usable and inclusive as they are adopted, adapted, and remixed by others.

In practice, accessibility is not a single feature but a set of intentional design choices, such as using clear structure, readable text, meaningful alternative text for images, and media that can be accessed in multiple ways. These choices benefit not only learners with disabilities but all students, particularly those accessing content across different devices and environments.

The criteria in this section outline key standards that support accessible OER. These standards are intended to guide constructive, improvement-focused feedback, helping reviewers identify strengths and opportunities to enhance access and usability.

These criteria are not intended to replace formal accessibility checks or technical evaluations. Instead, they complement tools such as accessibility checkers by focusing on how accessibility is experienced by learners and how it can be improved through design choices. Used together, these approaches support a more holistic understanding of accessibility in OER.

---

## C1: Text Structure and Screen Reader Navigation

1. All text content includes proper heading hierarchy (H1–H6), with logical nesting and no skipped levels.
2. Lists use proper markup (bulleted/numbered).
3. Reading order matches visual layout and intended flow of information.
4. Page navigation coding supports the use of adaptive technology.

## C2: Visual Design and Color

1. Text and background colors meet WCAG 2.1 AA contrast ratios (4.5:1 for normal text, 3:1 for large text).
2. Information is not conveyed by color alone.
3. Font size is minimum 12pt for body text.
4. Visual layout supports logical reading flow.

## C3: Alternative Text and Image Accessibility

1. All informative images include descriptive alternative text.
2. Decorative images are marked with null alt text.
3. Charts and graphs include data tables or detailed descriptions.

## C4: Multimedia Accessibility

1. Videos include accurate closed captions and transcripts.
2. Audio content provides complete transcripts.
3. Media players are keyboard accessible and compatible with assistive technology.
4. Auto-playing content can be paused or stopped.

## C5: Interactive Elements and Forms

1. All interactive elements (links, buttons, form fields) are keyboard accessible.
2. Form fields have labels and instructions.
3. Error messages are descriptive and actionable to promote troubleshooting.
4. Focus indicators are visible and logical.

## C6: Table Structure and Data Presentation

1. Data tables include proper header markup and scope attributes.
2. Tables are not used for layout purposes.
3. Complex tables provide summaries or navigation aids.
4. Row and column relationships are clearly defined.

## C7: Link Quality and Context

1. Link text is descriptive and meaningful out of context (in general, avoids "click here" or URL-only links unless links themselves are descriptive).
2. Links indicate when opening in new windows or leading to different file formats.
3. Link purposes are clear from context or link text alone.

## C8: Technical Format and Compatibility

1. Content is available in formats that preserve accessibility features.
2. Resources work across multiple devices and platforms (responsive design).
3. The learner does not need to use additional specific software or plugins to access resources.
